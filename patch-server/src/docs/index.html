Apa itu Moccipult?

Self-hosted code push untuk aplikasi Flutter — push update langsung ke HP user tanpa lewat Play Store.

⚡

Cepat

Push perbaikan bug dalam hitungan detik, bukan hari. Update otomatis saat user buka app.

🔒

Self-Hosted

Data tetap di server lo. Ga lewat pihak ketiga. Full kontrol.

📦

Ringan

Patch cuma mengandung bagian yang berubah, ukuran ratusan KB bukan puluhan MB.

💰

Gratis

Tanpa biaya subscription. Server Express + SQLite + S3 storage.

Quick Start

Dari nol sampai bisa push patch dalam 5 menit.

Step 1: Install Moccipult

📥

Installer (Recommended)

Download installer dari GitHub Releases. Otomatis install + tambah ke PATH.

🪟 Windowsmoccipult-setup-x.x.x.exe
🐧 Linuxmoccipult_x.x.x_amd64.deb
🍎 macOSmoccipult-x.x.x-macos.pkg
# Windows: double-click moccipult-setup.exe, next-next-finish

# Linux:
sudo dpkg -i moccipult_*.deb

# macOS:
open moccipult-*.pkg   # ikuti installer wizard
📦

Portable Binary

Download binary langsung, taruh mana aja. Tanpa install wizard.

🪟 Windowsmoccipult-windows.exe
🐧 Linuxmoccipult-linux

🍎 macOS ARMmoccipult-macos-arm

chmod +x moccipult-*
./moccipult-* status

# Windows:

Hubungkan CLI ke server Moccipult.

Integrasi Flutter

Cara menggunakan Moccipult di project Flutter kamu.

1. Tambahkan Package

Edit pubspec.yaml:

dependencies:
  moccipult:
    git:
      url: https://github.com/fatmuh/moccipult.git
      path: flutter/moccipult

Lalu jalankan:

flutter pub get

2. Tambahkan di main.dart

Ganti YOUR_APP_ID dan YOUR_SERVER_URL dengan data dari moccipult init.

import 'package:flutter/material.dart';
import 'package:moccipult/moccipult.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  // ── Tambahkan kode ini ──────────────────
  final updater = MoccipultUpdater(
    serverUrl: 'https://patches.yourdomain.com',
    appId: 'YOUR_APP_ID',     // dari moccipult init
    platform: 'android',      // atau 'ios'
    channel: 'stable',
  );
  await updater.checkAndApply();
  // ── Selesai ──────────────────────────────

  runApp(const MyApp());
}

class MyApp extends StatelessWidget {
  const MyApp({super.key});
  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'My App',
      home: const HomeScreen(),
    );
  }
}

class HomeScreen extends StatelessWidget {
  const HomeScreen({super.key});
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('My App')),
      body: const Center(child: Text('App berjalan!')),
    );
  }
}

3. Advanced — Dengan Callbacks

Kamu bisa monitor proses update dengan callbacks:

final updater = MoccipultUpdater(
  serverUrl: 'https://patches.yourdomain.com',
  appId: 'YOUR_APP_ID',
);

// Ketika patch tersedia
updater.onUpdateAvailable = (patch) {
  print('🎉 Patch #${patch.patchNumber} tersedia!');
  print('   Ukuran: ${patch.fileSize} bytes');
};

// Progress download
updater.onDownloadProgress = (downloaded, total) {
  final percent = (downloaded / total * 100).toStringAsFixed(0);
  print('📥 Download: $percent%');
};

// Patch berhasil diterapkan
updater.onPatchApplied = (patchNumber) {
  print('✅ Patch #$patchNumber berhasil diterapkan!');
};

// Error (TIDAK akan crash app)
updater.onError = (error) {
  print('⚠️ Update error: $error');
};

await updater.checkAndApply();

4. Cek Patch Number di UI

FutureBuilder<int>(
  future: MoccipultUpdater(
    serverUrl: 'https://patches.yourdomain.com',
    appId: 'YOUR_APP_ID',
  ).getCurrentPatchNumber(),
  builder: (context, snapshot) {
    return Text('Patch version: #${snapshot.data ?? 0}');
  },
)
✅
Aman untuk production. Jika server down, app tetap jalan normal. Error hanya di-log, tidak crash.

Setup Shorebird CLI + Moccipult

Fork Shorebird CLI open-source, patch supaya ngomong ke server Moccipult kamu. 100% gratis, tanpa batasan.

✅
Kenapa cara ini? Shorebird CLI open-source (MIT/Apache). Kita cuma ganti server URL-nya. Engine + compiler tetap resmi dari Shorebird. Server kita handle distribusi patch — jadi gratis & unlimited.

Manual Step-by-Step

Kalau mau ngerti prosesnya atau script error:

1

Clone Shorebird repos

mkdir shorebird-workspace && cd shorebird-workspace

# Clone CLI
 git clone --depth 1 https://github.com/shorebirdtech/shorebird.git

# Clone updater
 git clone --depth 1 https://github.com/shorebirdtech/updater.git
2

Patch semua URL ke server kamu

# Dari root repo moccipult
 python patch_repos.py \
   --shorebird-path ./shorebird-workspace/shorebird \
   --updater-path ./shorebird-workspace/updater \
   --target-url https://patches.yourdomain.com \
   --dry-run      # preview dulu

# Kalau sudah yakin, tanpa --dry-run
 python patch_repos.py \
   --shorebird-path ./shorebird-workspace/shorebird \
   --updater-path ./shorebird-workspace/updater \
   --target-url https://patches.yourdomain.com

Script akan scan semua file dan mengganti api.shorebird.dev, api.shorebird.cloud, dll ke URL server kamu.

3

Build updater (Rust)

cd shorebird-workspace/updater
cargo build --release
4

Build Shorebird CLI

cd shorebird-workspace/shorebird

# Install dependencies
dart pub get

# Compile ke binary
dart compile exe bin/shorebird.dart -o shorebird   # Linux/Mac
dart compile exe bin/shorebird.dart -o shorebird.exe  # Windows
5

Test koneksi

./shorebird --version

# Login (skip kalau server kamu tanpa auth)
./shorebird login
6

Done! 🎉

CLI Shorebird sekarang ngomong ke server Moccipult kamu, bukan shorebird.dev.

Workflow Harian

Cara kerja sehari-hari setelah setup selesai.

SEKALI SAJA: Release Pertama

Build app pertama kali dengan Shorebird engine, upload ke Play Store.

1

Masuk ke folder Flutter project

cd /path/to/my-flutter-app
2

Build dengan Shorebird

# Pastikan CLI Shorebird yang sudah di-patch ada di PATH
 export PATH="$PATH:/path/to/shorebird-workspace/shorebird"

# Build release pertama
 shorebird release android

# CLI akan:
#   1. Register app ke Moccipult server
#   2. Build APK dengan Shorebird engine
#   3. Upload baseline snapshot ke server
#   4. Generate file .aab / .apk

# Output:
# ✅ App registered: MyApp (a1b2c3d4-...)
# ✅ Release created: v1.0.0 #1 (x1y2z3w4-...)
# ✅ Built: build/app/outputs/bundle/release/app-release.aab
3

Upload ke Play Store

Upload file .aab ke Google Play Console seperti biasa.

File ini SUDAH mengandung Shorebird updater — jadi app bisa menerima patch.

SETIAP KALI ADA BUG FIX: Push Patch

1

Fix bug di kode

Edit file Dart seperti biasa. Fix bug yang mau di-patch.

2

Push patch

shorebird patch android

# CLI akan:
#   1. Compile Flutter code baru
#   2. Compare dengan snapshot sebelumnya
#   3. Generate .bin (binary diff)
#   4. Upload ke Moccipult server

# Output:
# ✅ Patch #2 uploaded (3.2 KB)
# ✅ Available at: https://patches.yourdomain.com/downloads/...
3

Selesai! 🎉

User buka app → otomatis download patch → bug fix langsung aktif.

Tanpa upload ke Play Store. Tanpa review. Tanpa tunggu.

Tutorial Play Store

Langkah lengkap dari development sampai app live di Play Store dengan Moccipult.

🤖
Android = 100% aman. Tidak ada batasan dari Google untuk code push. Banyak app besar menggunakan mekanisme serupa.

Tahap 1: Setup Project

1

Buat project Flutter

flutter create my_app
cd my_app
2

Tambahkan Moccipult ke pubspec.yaml

dependencies:
  moccipult:
    git:
      url: https://github.com/fatmuh/moccipult.git
      path: flutter/moccipult
3

Edit main.dart

import 'package:flutter/material.dart';
import 'package:moccipult/moccipult.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  await MoccipultUpdater(
    serverUrl: 'https://patches.yourdomain.com',
    appId: 'YOUR_APP_ID',
    platform: 'android',
  ).checkAndApply();

  runApp(const MyApp());
}
4

Registrasi app di Moccipult

moccipult config server https://patches.yourdomain.com
moccipult init
# Catat App ID dan Release ID!

Tahap 2: Build & Upload ke Play Store

5

Buat signing key

keytool -genkey -v -keystore release.keystore \
  -alias myapp -keyalg RSA -keysize 2048 -validity 10000
6

Build AAB (Android App Bundle)

flutter build appbundle --release

File output ada di build/app/outputs/bundle/release/app-release.aab

7

Upload ke Google Play Console

  1. Buka Google Play Console
  2. Buat app baru → isi detail
  3. Menu Release → Production → Create new release
  4. Upload file .aab
  5. Isi release notes
  6. Review & Release
8

Tunggu review

Google review biasanya 1–3 hari untuk app baru. Setelah approve, app live di Play Store.

Tahap 3: Push Patch (Setiap Ada Bug Fix)

9

Fix bug di kode

Edit kode Flutter seperti biasa.

10

Generate & upload patch

# Build patch file (sesuaikan dengan workflow tim kamu)
moccipult patches upload -r <RELEASE_ID> -f patch.bin
11

Patch langsung live! 🎉

Semua user yang sudah install app akan otomatis mendapat patch saat buka app. Tanpa upload ke Play Store lagi.

⚠️
Perhatian: Code push hanya untuk perbaikan bug kecil. Untuk perubahan besar (fitur baru, permission baru, native code changes), tetap harus upload versi baru ke Play Store.

Tips

✅ Aman untuk code push

Fix typo, perbaikan UI, perbaikan logic bug, update teks, dsb.

❌ Harus upload baru ke Store

Tambah permission baru, ubah native code, update SDK version, ubah app icon.

Tutorial App Store

Panduan integrasi Moccipult dengan iOS App Store.

🍎
iOS ada risiko. Apple Guideline 2.5.2 melarang download executable code baru. Meskipun jarang di-enforce, ada kemungkinan app ditolak. Gunakan dengan risiko sendiri.

Rekomendasi Penggunaan di iOS

✅

Aman

App internal (enterprise), TestFlight, app yang tidak dipublikasikan di App Store.

⚠️

Risiko

App publik di App Store. Apple mungkin menolak atau ban app. Riskanya rendah tapi ADA.

Tahap 1: Setup Project iOS

1

Pastikan Xcode & certificate sudah siap

# Cek Xcode
xcode-select --install

# Cek certificate di Keychain Access
# Apple Development: your@email.com
2

Tambahkan Moccipult (sama seperti Android)

// main.dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  await MoccipultUpdater(
    serverUrl: 'https://patches.yourdomain.com',
    appId: 'YOUR_APP_ID',
    platform: 'ios',  // ← ganti ke ios
  ).checkAndApply();

  runApp(const MyApp());
}
3

Registrasi app dengan platform ios

moccipult apps create -n "My App" -p com.example.app --platform ios

Tahap 2: Build & Upload ke App Store

4

Build IPA

flutter build ipa --release
5

Upload via Xcode atau Transporter

  1. Buka Xcode → Organizer
  2. Pilih archive yang baru
  3. Klik Distribute App → App Store Connect
  4. Atau gunakan app Transporter dari Mac App Store
6

Submit di App Store Connect

  1. Buka App Store Connect
  2. Pilih app → App Store → iOS App
  3. Pilih build yang di-upload
  4. Isi: screenshot, deskripsi, keywords, dsb.
  5. Klik Submit for Review
7

Tunggu review Apple

Apple review biasanya 1–2 hari. App baru bisa lebih lama (sampai 7 hari).

Tahap 3: Push Patch ke iOS

8

Upload patch seperti biasa

moccipult patches upload -r <RELEASE_ID_IOS> -f patch.bin

Proses sama persis seperti Android. User iOS otomatis dapat patch.

Strategi Aman untuk iOS

🔥 Rekomendasi: Dual Strategy

Gunakan pendekatan berbeda per platform:

void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  final updater = MoccipultUpdater(
    serverUrl: 'https://patches.yourdomain.com',
    appId: Platform.isAndroid ? 'ANDROID_APP_ID' : 'IOS_APP_ID',
  );

  if (Platform.isAndroid) {
    // Android: full code push
    await updater.checkAndApply();
  } else {
    // iOS: hanya cek, tampilkan notif untuk update dari App Store
    final result = await updater.checkForPatch(
      version: packageInfo.version,
    );
    if (result.available) {
      // Tampilkan dialog: "Update tersedia, download dari App Store?"
    }
  }

  runApp(const MyApp());
}
💡
Tips: Untuk app internal (enterprise distribution), kamu bisa menggunakan full code push di iOS tanpa khawatir. Batasan Apple hanya berlaku untuk app publik di App Store.
