Apa itu Moccipult?

Self-hosted code push untuk Flutter — push update langsung ke HP user tanpa lewat Play Store.

⚡ Cepat

Push perbaikan bug dalam hitungan detik. Update otomatis saat user buka app.

🔒 Self-Hosted

Data tetap di server lo. Ga lewat pihak ketiga. Full kontrol.

📦 Ringan

Patch cuma mengandung bagian yang berubah — ratusan KB, bukan puluhan MB.

💰 Gratis

Tanpa biaya subscription. Server Express + SQLite + S3.

Integrasi Flutter

Tambahkan Moccipult ke project Flutter kamu.

1. Tambahkan Package

Edit pubspec.yaml:

dependencies:
  moccipult:
    git:
      url: https://github.com/fatmuh/moccipult.git
      path: flutter/moccipult
flutter pub get

2. Tambahkan di main.dart

import 'package:flutter/material.dart';
import 'package:moccipult/moccipult.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();

  // ── Tambahkan 3 baris ini ──
  await MoccipultUpdater(
    serverUrl: 'https://patches.yourdomain.com',
    appId: 'YOUR_APP_ID',
    appVersion: '1.0.0',
    platform: 'android',    // atau 'ios'
  ).checkAndApply();
  // ── Selesai ──

  runApp(const MyApp());
}

3. Dengan Callbacks (Optional)

final updater = MoccipultUpdater(
  serverUrl: 'https://patches.yourdomain.com',
  appId: 'YOUR_APP_ID',
  appVersion: '1.0.0',
);

updater.onUpdateAvailable = (patch) {
  print('Patch #${patch.patchNumber} tersedia! (${patch.fileSize} bytes)');
};

updater.onDownloadProgress = (downloaded, total) {
  print('Download: ${(downloaded / total * 100).toStringAsFixed(0)}%');
};

updater.onPatchApplied = (patchNumber) {
  print('Patch #$patchNumber berhasil diterapkan!');
};

updater.onError = (error) {
  print('Update error: $error');  // TIDAK akan crash app
};

await updater.checkAndApply();
✅
Aman untuk production. Jika server down, app tetap jalan normal. Error hanya di-log, tidak crash.

Setup Shorebird CLI

Patch Shorebird CLI supaya ngomong ke server Moccipult kamu. 100% gratis, tanpa batasan.

💡
Konsep: Shorebird CLI itu open-source (Dart). Kita clone source-nya, ganti semua URL api.shorebird.dev jadi server Moccipult, lalu compile ulang. Engine & patch algorithm tetap resmi Shorebird.

Gambaran Besar

SEBELUM (Shorebird resmi):
  shorebird CLI ──▶ api.shorebird.dev ──▶ Shorebird Cloud (bayar)

SESUDAH (Moccipult):
  shorebird CLI ──▶ patches.yourdomain.com ──▶ Server kamu (GRATIS)
     (di-patch)          ↑                      ├── Express API
                     patch_repos.py             ├── SQLite DB
                                                └── S3 Storage

  Yang berubah: HANYA URL server
  Yang TETAP SAMA:
    ✅ Shorebird engine (modified Flutter engine)
    ✅ AOT snapshot compiler & patch generation
    ✅ Native updater code
    ✅ Dart/Flutter toolchain

Troubleshooting

❌ rustc: command not found

# macOS / Linux:
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# Windows:
# Download installer dari https://rustup.rs
# Atau: winget install Rustlang.Rustup

❌ protoc: command not found

# Windows (pilih salah satu):
choco install protobuf       # kalau ada Chocolatey
winget install protobuf      # kalau ada winget

# Atau download manual:
# https://github.com/protocolbuffers/protobuf/releases
# Download protoc-xx.x-win64.zip, extract, tambah bin/ ke PATH

# macOS:
brew install protobuf

# Ubuntu/Debian:
sudo apt install protobuf-compiler

❌ dart: command not found

Dart bundel dengan Flutter SDK. Pastikan flutter/bin ada di PATH.

❌ CLI jalan tapi masih nyasar ke shorebird.dev

Artinya patch belum ke-apply. Jalankan patch_repos.py tanpa --dry-run, lalu rebuild.

grep -r "shorebird.dev" ~/shorebird-workspace/shorebird/lib/ | head -5
# Kalau ada hasil → belum di-patch

Deploy ke App Store

Panduan integrasi dengan iOS App Store.

🍎
iOS ada risiko. Apple Guideline 2.5.2 melarang download executable code. Meskipun jarang di-enforce, ada kemungkinan app ditolak. Gunakan dengan risiko sendiri.

Rekomendasi

✅ Aman

App internal (enterprise), TestFlight, app yang tidak dipublikasikan di App Store.

⚠️ Risiko

App publik di App Store. Apple mungkin menolak. Riskanya rendah tapi ADA.

  serverUrl: 'https://patches.yourdomain.com',
  appId: Platform.isAndroid ? 'ANDROID_APP_ID' : 'IOS_APP_ID',
  appVersion: '1.0.0',
);

if (Platform.isAndroid) {
  // Android: full code push
  await updater.checkAndApply();
} else {
  // iOS: hanya cek, tampilkan notif update dari App Store
  final result = await updater.checkForPatch(version: version);
  if (result.available) {
    // Tampilkan dialog: "Update tersedia di App Store"
  }
}
